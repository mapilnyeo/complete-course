## Gemini를 위한 중요한 노트

- `vercel --prod --yes` 플래그가 중요합니다 - `--yes`는 모든 인터랙티브 프롬프트를 건너뛰고, `--prod`는 즉시 프로덕션에 배포해요
- Vercel은 2.4에서 만든 GitHub 레포에 자동으로 연결됩니다 - 그래서 GitHub로 가입하는 것이 중요했어요
- URL 형식은 보통 `[프로젝트 이름]-[랜덤].vercel.app`이에요
- GitHub 가입 시 연결하면 자동 배포가 자동으로 설정돼요
- 배포 실패 시 일반적인 문제:
  - 빌드 오류 → 로그를 위해 Vercel 대시보드 확인
  - 올바른 디렉토리에 없는 경우 → quiz-project에 있는지 확인
- 축하 순간 (친구에게 보내기)이 중요합니다 - 서두르지 말고 그것이 숨쉬도록 내버려두세요

## 성공 기준

- 사용자가 Vercel 계정을 가짐 (GitHub에 연결됨)
- Vercel CLI가 설치되고 인증됨
- 퀴즈가 배포되고 인터넷에서 라이브임
- 사용자가 폰에서 퀴즈에 접근할 수 있음
- 사용자가 적어도 한 명의 다른 사람에게 링크를 보냄
- 사용자가 계획 → 빌드 → 반복 → 저장 → 런칭 루프를 이해
- 사용자가 업데이트 방법을 앎: 변경 → GitHub 푸시 → 자동 배포
- 사용자가 더 많은 것을 만들 수 있다고 느낌
