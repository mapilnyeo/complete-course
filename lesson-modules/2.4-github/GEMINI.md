브라우저가 로그인하도록 열릴 수 있어요 - 그냥 안내에 따르고 완료되면 돌아오세요.

**Gemini CLI 팁:** GitHub 인증 설정에는 `/setup-github`를 빠른 방법으로 사용할 수 있어요!

ACTION: GitHub CLI 설정:

1. 이미 설치됐는지 확인:
```
gh --version
```

2. 설치 안 됐다면:
   - Mac에서는: `brew install gh`
   - Windows에서는: `winget install --id GitHub.cli`
   - 이것들이 실패하면: https://cli.github.com에 가서 수동으로 다운로드하고 돌아오세요

3. 설치됐으면 인증:
```bash
gh auth login --web --git-protocol https
```

4. 인증을 위한 브라우저 안내를 따르세요.

5. 사용자에게 말하기: "브라우저에서 안내에 따라 인증해주세요. 완료되면 알려주세요."

ACTION: 순서대로 다음 명령어 실행:

1. quiz-project 폴더로 이동하고 git 초기화 (아직 안 했다면):
```

```

2. 모든 파일 추가:
```

```

3. 초기 커밋 만들기:
```
git commit -m "초기 커밋 - 커피 성격 퀴즈"
```

4. GitHub 레포 만들고 푸시:
```
gh repo create quiz-project --private --source=. --push
```

완료! 이제 코드가 GitHub에 저장됐어요. 개발자들은 이것을 "푸시"라고 해요 - 클라우드로 올리는 것이에요.

## Gemini를 위한 중요한 노트

- GitHub CLI (gh)가 인증을 처리하는 가장 쉬운 방법입니다 - 사용자에게 SSH 키 설정을 요청하지 마세요
- `--private` 플래그가 중요합니다 - 사용자는 기본적으로 퀴즈 코드가 공개되는 것을 원하지 않아요
- quiz-project라는 레포 이름이 이미 있으면 gh에서 오류가 납니다 - 다른 이름을 선택하도록 도와주세요
- `gh repo view --web` 명령어가 URL을 직접 구성하려는 것보다 훨씬 깔끔해요
- `/setup-github`는 GitHub 설정에 도움이 되는 Gemini CLI 내장 명령어예요
- Next.js 프로젝트에는 이미 .gitignore가 있어서 node_modules가 커밋되지 않아요

## 성공 기준

- 사용자가 GitHub 계정을 가짐
- GitHub CLI가 설치되고 인증됨
- 코드가 비공개 GitHub 레포에 커밋되고 푸시됨
- 사용자가 github.com에서 레포를 볼 수 있음
- 사용자가 미래 변경사항 저장을 위해 "GitHub에 푸시해줘"라고 요청할 수 있다는 것을 이해
