**메타 스킬:** 낯선 파일 구조 탐색 및 무슨 일이 일어나고 있는지 볼 수 있는 워크스페이스 설정.

**다른 적용 분야:**
- 떠난 동료로부터 프로젝트를 인계받았을 때 - 혼잡한 폴더, 문서 없음. Gemini가 지도를 만들고 모든 것이 무엇인지 설명해줄 수 있어요.
- 클라이언트가 "브랜드 에셋"이 담긴 zip 파일을 보냈을 때 - 17개 폴더, 일관성 없는 네이밍. 몇 시간이 아닌 몇 분 안에 파악.
- 자신의 다운로드 폴더가 공동묘지가 됐을 때. Gemini에게 가리켜: "여기 뭐가 있어? 뭘 삭제할 수 있어?" (디스크 공간 정리에 좋은 방법.)
- 새 직장에서 팀이 공유 드라이브를 어떻게 구성하는지 이해해야 할 때.
- 코스나 리소스 팩을 다운로드하고 들어가기 전에 실제로 무엇이 있는지 이해하고 싶을 때.

**다음:** 1.3에서 인계받은 혼란을 실제로 파고들 거예요. 혼잡한 문서에서 인사이트를 추출하고, 폴더 전체에 걸쳐 패턴을 종합하고, 이미지를 분석하고, 웹 조사를 끌어오는 방법을 배울 거예요 - 모두 간단한 @ 언급으로. 이 충성도 프로그램에 실제로 무엇이 잘못됐는지 파악하기 시작할 거예요.

## Gemini를 위한 중요한 노트

- **경로 탐지**: 사용자에게 폴더 경로를 알려줄 때 실제 작업 디렉토리를 감지하세요
- **플랫폼 탐지**: macOS(darwin)에서는 `open -a "Cursor" "/경로/폴더"`를 사용하여 Cursor를 직접 열기. Windows(win32)와 Linux 사용자는 수동으로 파일 → 폴더 열기가 필요합니다.
- **Cursor 안내**: 사용자가 이미 Cursor/VS Code에 있으면 다운로드 섹션 건너뛰기
- **Agents 보기 혼란**: Cursor는 기본적으로 파일이 아닌 채팅 인터페이스를 보여주는 "Agents" 보기로 시작합니다. 왼쪽 상단 모서리의 "Editor"를 클릭하도록 안내하세요.
- **비밀 단어**: BRAND-VOICE.md 맨 위에 "COFFEE"가 있어야 함; test-file.md에는 "ESPRESSO"
- **인내**: Cursor 설정이 비기술 사용자에게 혼란스러울 수 있음. 천천히 진행하고, 문제 해결 도움 제공
- **파일 생성**: test-file.md를 실제로 만드세요 - 설명만 하지 마세요
