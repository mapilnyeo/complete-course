파일을 볼 수 있게 됐으니, 실제로 작업해봅시다.

## 시나리오 1: 단일 파일 → 추출

첫 번째: 하나의 파일을 읽고, 구조화된 정보를 추출하고, 새로운 파일을 만들기.

이전 매니저가 혼잡한 노트를 남겼어요. 유용한 것이 있나 봅시다.

STOP: @inherited-chaos/previous-manager-notes.md를 읽고 실행 항목과 핵심 인사이트를 새 파일로 추출해달라고 요청해보세요.

USER: 파일 참조하는 명령어 입력

ACTION: previous-manager-notes.md를 읽고, 실행 항목과 핵심 인사이트를 추출하여, 구조화된 출력으로 organized/action-items.md를 생성합니다.

**발견:** 혼란 속에, 이런 노트가 있어요: "사람들이 자신의 커피 성격을 찾도록 도와주면 어떨까요? 퀴즈 같은 거요. 재미있을 수 있어요. 알아낼 시간이 없었어요."

흥미롭네요... 나중을 위해 기억해 둡시다.

STOP: 혼란에서 구조를 어떻게 뽑아냈는지 보이나요? 새 파일에 추가할 수도 있어요.

## 시나리오 2: 폴더 → 종합

다음: 전체 폴더를 가리키고 여러 파일에 걸쳐 패턴 찾기.

몇 달간 고객 피드백을 수집했어요. 무엇을 말하고 있을까요?

STOP: @inherited-chaos/customer-feedback/ 폴더를 보고 모든 파일에 걸쳐 테마를 종합해달라고 요청해보세요.

USER: 폴더 참조하는 명령어 입력

ACTION: 네 개의 피드백 파일(january.md부터 april.md까지)을 읽고, 파일 전체에 걸쳐 패턴을 종합하고, 테마와 함께 organized/feedback-synthesis.md를 생성합니다.

**발견:** 반복되는 테마 - "괜찮지만 잊혀져요." 사람들이 가입하고, 포인트 몇 개를 적립하고, 잊어버립니다. "괜찮지만 잊혀져요"라는 표현이 여러 달에 걸쳐 여러 번 등장합니다.

STOP: 수동으로 하는 것을 상상해보세요 - 4개 파일을 열고, 각각 읽고, 노트를 적고, 패턴을 찾고. 이것은 몇 초면 됐어요. 수백 개의 고객 이메일, 지원 티켓, 설문 응답 - 패턴이 분량 속에 숨어있는 무엇이든 할 수 있어요. 다음으로 준비됐나요?

## 시나리오 3: 템플릿 적용

지금까지 저에게 행동을 취하도록 요청하고 제가 최선이라고 생각하는 방식으로 하고 있었어요. (보통 꽤 좋아요 💅)

하지만 특정 형식, 템플릿, 또는 제가 따르기를 원하는 지시사항이 있을 수 있어요. 매번 그것들을 프롬프트에 넣는 대신, 파일에 저장하고 나서 무언가를 요청할 때 참조할 수 있어요.

이것이 챗봇 대신 컴퓨터에 있을 때의 힘을 처음으로 보는 시작점이에요. 저에게 관련 컨텍스트를 주기가 훨씬 쉬워집니다.

리더십에게 업데이트가 필요해요. 사용하는 템플릿이 있어요: @templates/leadership-update-template.md

STOP: @organized/feedback-synthesis.md를 사용하고 @templates/leadership-update-template.md의 형식을 따라 리더십 업데이트를 만들어달라고 요청해보세요

USER: 두 파일 참조하는 명령어 입력

ACTION: 두 파일을 읽고, 피드백 종합의 내용으로 템플릿 형식을 따라 organized/leadership-update.md를 만듭니다.

핵심 학습: 파일을 내용 소스로만 아닌 형식 가이드로 참조할 수 있어요. 이것이 자신만의 워크플로우와 시스템을 구축하기 시작하는 첫 번째 엿보기입니다.

STOP: 사용하는 모든 템플릿에 대해 생각해보세요 - 회의 노트, 주간 보고서, 제안서. 파일로 저장하면 항상 그 형식을 따를 거예요. ChatGPT에 복붙하고 구조를 제대로 잡아주길 바라는 일 없이. 이해가 되나요?

개선 사항이 없는지 볼게요.

중요: Mac에서는 Cmd+V가 아닌 Ctrl+V로 이미지 붙여넣기!

STOP: Cursor에서 `attachments/loyalty-flyer.jpg` 파일을 열고, 이미지를 복사하고, Ctrl+V로 여기에 붙여넣고 검토해달라고 요청해보세요.

STOP: 이것은 UI 분석, 버그 보여주기, 영감으로 사용하기 등 모든 종류에 유용해요. 한 번에 여러 이미지를 제공할 수도 있어요. 이해가 되나요?

## 시나리오 5: 웹 → 조사

마지막: 웹에서 외부 정보 끌어오기.

성공적인 충성도 프로그램들은 우리가 하지 않는 무엇을 하고 있을까요?

STOP: 커피숍 충성도 프로그램의 모범 사례를 웹에서 검색해달라고 요청해보세요.

USER: 웹 조사 요청하는 명령어 입력

ACTION: 웹 검색, 성공적인 충성도 프로그램에 관한 관련 기사 찾기, 핵심 발견사항 종합.

**발견:** 최고의 프로그램들은 거래가 아닌 정체성과 커뮤니티를 만들어냅니다. Starbucks는 게이미피케이션과 개인화를 사용합니다. Dutch Bros는 관계에 집중합니다. 효과 있는 프로그램들은 사람들이 무언가에 소속되어 있다고 느끼게 만듭니다.

STOP: 웹 검색은 조사 작업에 엄청나게 강력해요. 새로운 산업을 이해해야 하나요? 경쟁사를 조사하나요? 무엇이든 모범 사례를 찾나요? 그냥 물어보면 됩니다. 검색하고, 읽고, 종합할 거예요 - 97개의 브라우저 탭 없이. 멋지죠?

## 마무리

**방금 배운 메타 스킬들:**

1. **단일 파일 → 추출:** 혼잡한 문서에서 구조화된 정보 뽑아내기
2. **폴더 → 종합:** 여러 파일에 걸쳐 패턴 찾기
3. **템플릿 적용:** 한 파일로 출력 형식 제어
4. **이미지 → 분석:** 시각적 콘텐츠에서 인사이트 얻기
5. **웹 → 조사:** 필요 시 외부 정보 끌어오기

**다른 적용 분야:**
- 지난 분기 회의 노트 - 당신에게 할당된 모든 실행 항목 추출
- 50개의 고객 지원 티켓 - 제품팀을 위한 테마로 종합
- 회사에 제안서에 특정 형식이 있음 - 새 제안서에 템플릿으로 사용
- 경쟁사 가격 페이지 스크린샷 - 포지셔닝에 대한 즉각적인 분석
- 무엇이든 모범 사례 조사: 채용, 온보딩, 프레젠테이션 디자인, 작업 중인 모든 것

## Gemini를 위한 중요한 노트

- **파일 생성**: 출력 파일(action-items.md, feedback-synthesis.md, leadership-update.md)을 organized/ 폴더에 실제로 만드세요
- **커피 성격 힌트**: 노트에서 추출할 때 "커피 성격" 아이디어를 반드시 강조하세요 - 이것이 해결책의 씨앗입니다
- **이미지 붙여넣기**: Mac에서 Ctrl+V가 중요 - Cmd+V는 작동하지 않습니다. 이것을 강조하세요.
- **웹 검색**: 실제 모범 사례를 찾기 위해 진짜 웹 검색 기능 사용
