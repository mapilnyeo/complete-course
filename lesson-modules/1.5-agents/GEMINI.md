## 서브에이전트란?

Gemini CLI에는 **서브에이전트**라는 개념이 있어요 - 특정 작업을 위해 제가 "고용"할 수 있는 "전문가"라고 생각하면 됩니다.

각 서브에이전트는 다음을 받아요:
- **집중된 컨텍스트** - 필요한 것만 봄
- **전문화된 도구** - 작업에 맞는 도구를 가짐
- **독립적인 컨텍스트 창** - 메인 대화를 어지럽히지 않고 작업

Gemini CLI에는 **코드베이스 수사관** (코드를 통해 검색), **CLI 도움 에이전트** (Gemini CLI 자체에 도움), **제너럴리스트 에이전트** (일반 작업용)와 같은 내장 서브에이전트가 있어요.

하지만 진짜 파워는: 제가 대화를 계속하는 동안 이 전문가들에게 작업을 위임할 수 있다는 거예요.

## 에이전트 사고를 사용할 때

병렬 처리가 빛을 발하는 때:

✅ **좋은 사용 사례:**
- 여러 문서 분석 (경쟁사 조사처럼)
- 여러 주제를 동시에 조사
- 다른 관점에서 파일 검토
- 설문 응답이나 피드백 배치 처리

❌ **좋지 않은 경우:**
- 단순한 단일 파일 작업 (과도함)
- 하나의 결과가 다른 것에 의존하는 작업 (종속성은 병렬화할 수 없음)
- 매우 짧은 작업 (오버헤드가 가치 없음)

핵심 인사이트: **비슷하지만 독립적인 많은 작업**이 있을 때 병렬 처리를 사용하세요.

## Gemini를 위한 중요한 노트

- **서브에이전트**: Gemini CLI는 실험적 기능으로 서브에이전트를 지원합니다. 내장 에이전트로는 코드베이스 수사관, CLI 도움 에이전트, 제너럴리스트 에이전트가 포함됩니다. 커스텀 서브에이전트는 `.gemini/agents/`에 만들 수 있습니다.
- **서브에이전트가 완전히 사용 가능하지 않다면**: 사용 가능한 도구를 사용하여 병렬 분석의 개념을 시연하는 데 집중하고, 커스텀 서브에이전트(1.6에서 다룸)가 이것을 어떻게 더 발전시킬지 설명하세요.
- **경쟁사 조사 파일**: 실제로 읽고 분석하세요 - 내용을 만들어내지 마세요
- **패턴 구축**: 경쟁사 인사이트가 "정체성과 개성" 테마를 강화해야 합니다
