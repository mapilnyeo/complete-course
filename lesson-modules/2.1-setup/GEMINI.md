## PM/엔지니어 마인드셋

전통적인 소프트웨어 개발에서는 두 가지 다른 역할이 있어요:
- **프로덕트 매니저 (PM)**: 사용자가 무엇이 필요한지, 앱이 무엇을 해야 하는지 결정
- **엔지니어**: 그것을 코드로 구현

AI가 있으면 둘 다가 됩니다. 직접 코드를 쓸 필요는 없지만 "무엇"을 "어떻게"만큼이나 명확하게 정의해야 해요.

두 가지를 다루는 **계획 → 빌드 → 반복 → 저장 → 배포** 루프가 있어요:
- **계획**: 요구사항 정의 (이것이 PM 역할)
- **빌드**: Gemini가 코드를 만듦 (엔지니어 역할)
- **반복**: 보고 피드백을 주고 개선 (다시 PM)
- **저장**: GitHub 백업 (책임감 있는 소유자)
- **배포**: 인터넷에 올리기 (완성!)

## 기술 스택

Gemini가 앱을 만들 거예요. 어떤 기술을 사용할지 알려줘야 해요. 우리는 이것을 사용할 거예요:

- **JavaScript** - 웹을 구동하는 언어. 가장 인기 있는 코딩 언어.
- **Next.js** - JavaScript 프레임워크. 웹앱을 더 쉽게 만들어줌. 이걸 "비계"(scaffold)라고 생각해요 - 앱을 위한 구조.
- **Vercel** - 인터넷에 Next.js 앱을 배포하는 서비스. Next.js를 만든 같은 회사 제품이라 완벽하게 작동.

이것들을 알 필요가 없어요. Gemini가 다 처리할 거예요. 그냥 이것들이 존재한다는 것을 알아두세요.
